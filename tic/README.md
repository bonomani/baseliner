# TIC

TIC is the Test Intent Contract: a generic framework for specifying tests in a
clear, repeatable, and contract-aligned way.

Use TIC when you want to describe:

- test intent
- test structure
- setup, action, observation, and assertions
- traceability to a requirement or contract
- reporting rules for pass, fail, skip, and diagnostics

TIC is framework-neutral and complements:

- UIC for pre-flight intent and validation
- UCC for execution and convergence semantics

The canonical specification lives in [`SPEC.md`](./SPEC.md).
